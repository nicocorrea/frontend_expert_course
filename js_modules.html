Modules

Check on the console in web-dev-tools and you will see the outputted text, and no 'import' was called, meaning that we added all our variables to the global namespace, which is not good. We want them to be accessible within their file rather than globally, and then we import them individually.

Otherwise, using global namespace can case naming conflicts.

A couple of remarks for when using modules:

  • The file will be imported in strict mode by default
  • The await keyword can be used at the top level
  • The script will be deferred by default
  • The this will be undefined whereas in standard code this will be the window.
